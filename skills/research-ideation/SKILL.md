# Research Ideation for Empirical Economics

Use this skill for the front end of an empirical economics project.

## Core Workflow

```text
Question framing -> Literature map -> Identification strategy -> Data feasibility -> Research design
```

## Standard Outputs

- `literature-review.md`
- `identification-map.md`
- `data-source-plan.md`
- `research-design.md`
- `references.bib` when sources are verified

## Working Rules

- make the question empirically answerable, not merely interesting
- state the source of identifying variation explicitly
- link each proposed contribution to a specific literature gap or unresolved disagreement
- prefer concrete data plans over vague promises to "collect data later"
- treat working papers as important but track publication status carefully

## What To Clarify

For each project, specify:

- treatment or exposure
- outcome
- unit of observation
- time horizon
- identifying variation
- likely threats to identification
- feasible data sources
