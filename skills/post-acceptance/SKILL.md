# Post-Acceptance Workflow for Empirical Economics

Use this skill after an economics paper is accepted or reaches a stable internal-share stage.

## Default Deliverables

- replication package cleanup
- appendix and online appendix finalization
- data and code availability statement
- journal-ready table and figure inventory
- seminar or workshop slides
- policy brief or non-technical summary when relevant

## Workflow

```text
Accepted or near-final paper -> lock manuscript and appendix structure -> clean replication package -> check tables and provenance -> prepare disclosure language -> prepare seminar or policy materials if needed
```

## Replication Package Standard

Verify:

- raw, processed, and final analysis files are separated
- do-file execution order is documented
- table and figure provenance is explicit
- logs exist for major analysis steps
- path placeholders are clean and portable
- confidential or non-shareable data are clearly documented

## Seminar And Policy Communication

If the user asks for dissemination materials, prioritize:

- seminar slide outline
- workshop presentation deck
- policy brief
- short non-technical summary

Keep the tone economics-journal and policy-facing.
