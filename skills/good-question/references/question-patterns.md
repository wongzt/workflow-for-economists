# Question Strengthening Patterns

Use this card after raw candidate generation, especially when the output still sounds like a topic, gap, method, or project activity rather than a research question.

Sources synthesized from the existing cards: problem choice, problem picking, strong inference, problematization, question development, research taste, and Orchestra-style ideation lenses.

## Core Idea

Most weak questions fail because they are not yet questions. They are often topics, methods, settings, or ambitions. Convert them into questions by naming the uncertainty, the stake, the rival explanations, and the evidence that could change belief.

## Rewrite Patterns

| Weak form | Stronger form |
|---|---|
| "Study X in context Y" | "Under what boundary conditions does X change in Y, and what would that imply?" |
| "Use method M to analyze X" | "What uncertainty about X can M resolve better than simpler evidence?" |
| "No one has studied X" | "Which assumption in the existing literature becomes questionable if X behaves differently?" |
| "Does X affect Y?" | "Which mechanism linking X to Y best survives rival explanations and measurement artifacts?" |
| "Improve performance on benchmark B" | "What failure mode of current methods does improvement on B actually diagnose?" |
| "Apply theory T to setting S" | "What prediction does T make in S that competing theories would not make?" |
| "Explore recent trend Z" | "What changed recently that makes Z newly attackable or newly important?" |

## Strengthening Procedure

1. Label the weak form: topic, gap, method, effect, mechanism, benchmark, or application.
2. Name the missing stake: theory, method, practice, policy, or next experiment.
3. Add a rival: an alternative explanation, baseline, boundary condition, or null.
4. Add a falsifier: what observation would weaken the question's favored direction?
5. Shrink scope until a two-week pilot could update belief.

## Output Format

```markdown
**Before:** ...
**Weak form:** topic / gap / method / effect / mechanism / benchmark / application
**After:** ...
**What changed:** ...
**First falsifier:** ...
```

## Red Flags

- The rewrite only makes the sentence longer.
- The new question still depends on one favorite method.
- The stake is "publishability" rather than knowledge gained.
- No plausible answer would change the user's next action.
