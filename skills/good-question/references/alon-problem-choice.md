# Alon: Problem Choice

Use this card when choosing among research directions, judging whether a project is worth doing, or helping a researcher escape a stuck project.

Source: Uri Alon, "How To Choose a Good Scientific Problem", Molecular Cell, 2009. PDF: https://www.weizmann.ac.il/mcb/UriAlon/sites/mcb.UriAlon/files/uploads/nurturing/howtochoosegoodproblem.pdf

## Core Idea

Map candidate problems by both interest and feasibility. A good research problem is not simply novel; it should sit near the frontier where the researcher can learn something important with available tools, time, and motivation.

Treat confusion during a project as potentially productive. A project may enter a "cloud" where the original problem stops working; help the researcher notice whether a better question has appeared.

## Procedure

1. List 3-7 candidate problems or variants.
2. Score each on interest and feasibility from 1-5.
3. Ask what would increase feasibility without making the question trivial.
4. Ask what would increase interest without making the project impossible.
5. Identify the frontier: candidates that are not dominated by another candidate on both dimensions.
6. For the top candidate, name the smallest pilot that can update belief within two weeks.

## Prompts

- Who would care if this question were answered?
- What would become possible, clearer, or impossible if the result were known?
- Is the problem hard because it is deep, or hard because access and logistics are poor?
- What part of the question can be shrunk without losing the intellectual stake?
- If the current project fails, what unexpected observation might become the real question?

## Red Flags

- High feasibility but low interest: likely a routine exercise.
- High interest but near-zero feasibility: park it unless a new tool, data source, or collaborator changes the map.
- The user is attached to a method, but cannot name the problem it solves.
- The "important" claim only means fashionable, not consequential.
