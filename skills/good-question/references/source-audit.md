# Source Audit

Use this card when the user asks for current literature, latest work, field-specific grounding, "deep research", reviewer expectations, target journals, or any claim that depends on the state of a field.

## Core Idea

Sources are not decoration. A source only counts if it supports the specific claim being made. Do not let citations become a confidence costume.

## Audit Procedure

1. List the field claims that matter for choosing the question.
2. For each claim, attach the strongest available source.
3. Label source type: primary study, review, benchmark, guideline, editorial, dataset/tool release, lab post, preprint, or secondary summary.
4. Mark support strength:
   - **Direct:** source explicitly supports the claim.
   - **Partial:** source supports part of the claim; narrow the wording.
   - **Context only:** useful background but not evidence for the claim.
   - **Unsupported:** remove or rewrite the claim.
5. Separate "not found" from "does not exist."
6. Convert audited evidence into question opportunities, not a literature list.

## Required Repairs

- **Unsupported literature gap:** rewrite as "I did not verify this gap yet" or ask for a brief search.
- **Overbroad consensus:** replace with "several sources suggest..." or name the disagreement.
- **Preprint-only claim:** mark as provisional unless the claim is only about the preprint's existence.
- **Trend claim:** state what changed recently and cite sources that show the change.
- **Reviewer expectation:** source from journal aims, calls for papers, editorials, guidelines, or prior reviews.

## Red Flags

- A citation is attached to a sentence it does not actually support.
- A review is used to claim a new empirical result.
- A single paper is treated as field consensus.
- An abstract-only skim becomes a strong claim.
- The agent says "nobody has studied X" because search did not immediately find X.
- The question follows what is fashionable but cannot name what answering it changes.
